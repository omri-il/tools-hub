מחולל אימייל אוטומטי לטופס

כותבים את המכתב פעם אחת, מקבלים קוד מוכן — וכל מי שממלא את טופס ה-Google שלכם מקבל אותו מיד, אוטומטית. בלי לכתוב שורת קוד.

א.

תוכן המכתב

🎨 עיצוב ושולח — צבע, גופן, לוגו, חתימה
או הדביקו קישור למעלה

⚠️ הקובץ גדול יחסית — הקוד שנוצר יהיה ארוך. כדאי להעלות תמונה קטנה או פשוטה יותר.

אפשר להדביק קישור ישיר לתמונה (אתר / Google Drive) או להעלות קובץ מהמחשב — קובץ שתעלו מוטמע ישירות בקוד כדי שיוצג בכל תוכנת דואר. יופיע במרכז בראש המייל.

הדביקו קישור לקובץ ב-Drive (PDF / מצגת / תמונה). שתפו אותו כ"כל מי שיש לו הקישור — צופה". הקובץ יצורף לכל מייל. הקוד ישתמש ב-DriveApp ולכן יבקש הרשאת Drive בפעם הראשונה.

⚙️ אפשרויות מתקדמות פעיל — עמודת אימייל, פנייה בשם, מעקב

פותחים את גיליון התגובות, מסמנים את שורת הכותרות (ואם רוצים — גם שורת תגובה אחת), מעתיקים ב-Ctrl+C ומדביקים כאן. הכלי יזהה לבד את עמודת האימייל והשם ויעדכן את הבחירות למטה. הכול נשאר בדפדפן שלכם.

עמודה A היא תמיד חותמת הזמן. אם הפעלתם "Collect email addresses" בטופס — האימייל בעמודה B, השאירו כברירת מחדל. אם ביקשתם את האימייל כשאלה רגילה בטופס — פתחו את הגיליון, בדקו באיזו עמודה מופיעה כתובת המייל, ובחרו אותה כאן.

פתחו את הגיליון ובדקו באיזו עמודה מופיע השם, ובחרו אותה כאן. (עמודה A = חותמת זמן; אם האימייל נאסף אוטומטית הוא בעמודה B, והשאלה הראשונה בטופס היא עמודה C.)

🔒 אם תמלא — הקוד יוסיף משתתפים אוטומטית למעקב 5 הימים. הכתובת נשמרת רק במחשב שלך ולא מתפרסמת בדף.

🔗 כל כפתור במייל יעבור דרך מעקב, והלחיצה תירשם בעמודות מעקב לצד תשובות הטופס (לחצו? · כפתור אחרון + זמן · מספר לחיצות), וימשיך מיד ליעד. צריך לפרסם את הסקריפט כ-Web App ולהדביק את הכתובת — ההוראות יופיעו מתחת לקוד.

ג.

הקוד המוכן להדבקה



      

        איך מתקינים את הקוד? שלושה שלבים
        
    
          
  1. ודאו שהטופס אוסף כתובות אימייל — או דרך "Collect email addresses" (אז האימייל בעמודה B), או כשאלה רגילה בטופס. אם האימייל אינו בעמודה B, בחרו את העמודה הנכונה תחת ⚙️ אפשרויות מתקדמות → עמודת האימייל — או פשוט הדביקו שם שורה מהגיליון תחת 🔎 זיהוי אוטומטי והכלי יזהה לבד.
    2. 
          
  2. פתחו את הגיליון המחובר לטופס → תפריט Extensions → Apps Script. מחקו את הקוד הקיים, הדביקו את הקוד מכאן, ולחצו על 💾 לשמירה.
    3. 
          
  3. בחרו למעלה את הפונקציה setup ולחצו ▶ Run פעם אחת, ואשרו את ההרשאות. זהו — setup מקימה את הטריגר לבד, שולחת אליכם מייל בדיקה, ומודיעה לכם שהכול מוכן. אין צורך להגדיר טריגרים ידנית.
    4. 
        

        
זהו. מעכשיו כל מי שימלא את הטופס יקבל את המכתב אוטומטית.

      


      

        
📊 מעקב קליקים מופעל. הרצת setup כבר מוסיפה עבורכם עמודות מעקב לצד תשובות הטופס (לחצו? · כפתור אחרון + זמן · מספר לחיצות). נשאר צעד ידני אחד בלבד — לפרסם את הסקריפט כ-Web App ולהדביק את הכתובת, כדי שהלחיצות יירשמו. עשו זאת שלב אחר שלב:

        
    
          
  1. למעלה מימין לחצו על הכפתור Deploy.
    2. 
          
  2. בתפריט שנפתח בחרו New deployment.
    3. 
          
  3. ליד הכותרת "Select type" לחצו על גלגל השיניים ⚙️ ובחרו Web app.
    4. 
          
  4. בשדה Execute as בחרו Me (זה אתם).
    5. 
          
  5. בשדה Who has access בחרו Anyone (כל אחד).
    6. 
          
  6. לחצו על Deploy, ואם תתבקשו — אשרו את ההרשאות.
    7. 
          
  7. בחלון שנפתח תופיע כתובת Web app URL שמסתיימת ב-/exec — לחצו על כפתור ה-Copy שלצד הכתובת עצמה (לא להעתיק משורת הכתובת של הדפדפן).
    8. 
          
  8. חזרו לעורך, הדביקו את הכתובת בין המרכאות בשורה var WEBAPP_URL = ""; שבראש הקוד, ולחצו 💾 לשמירה.
    9. 
          
  9. הריצו שוב את setup — אמורה להופיע ההודעה "✅ מעקב קליקים פעיל". שלחו לעצמכם מייל בדיקה חדש ולחצו על כפתור — הפעם הוא יעביר אתכם ליעד.
    10. 
        

        
💡 זהו הצעד היחיד ש-Google לא מאפשרת לסקריפט לעשות לבד — לכן צריך את הלחיצות האלה פעם אחת.

        🛟 קיבלתם "Sorry, unable to open the file"? כמעט תמיד הכתובת שהודבקה כללה קטע כמו /u/1/ (כתובת שנעולה לחשבון שלכם). הכלי מנקה אותו אוטומטית — פשוט העתיקו מחדש את הקוד המעודכן מכאן, הדביקו בעורך, הדביקו שוב את כתובת ה-/exec, והריצו setup.

        🔁 אם בעתיד תשנו את הקוד — פרסמו גרסה חדשה (Deploy ← Manage deployments ← ✏️ ← New version), אחרת הקוד הישן ימשיך לרוץ.

        ℹ️ מייל הבדיקה שנשלח אליכם עשוי להוסיף בגיליון שורה אחת בתחתית עם האימייל שלכם — זה תקין.

      


      

        📨 אימייל מעקב מופעל. אין מה להגדיר — הרצת setup כבר מקימה את הבדיקה היומית ויוצרת בסיס (מי שכבר מילא לא יקבל מעקב למפרע; רק מילויים חדשים מכאן ואילך).

        💡 רוצים לראות איך המעקב ייראה? הריצו sendFollowUpTest — יישלח אליכם עותק בדיקה של כל מייל מעקב.
